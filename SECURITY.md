# 安全与隐私报告

## 支持范围

当前维护版本为最新 Beta 版本。较早版本可能不会单独提供安全修复。

## 报告方式

仓库上线后，请优先使用 GitHub 的私有漏洞报告功能。报告内容应包含复现步骤、受影响版本和影响说明，但不要附带客户项目、公司载体、真实测序文件、订购信息或个人联系方式。

如问题只能通过敏感样本复现，请先提供人工构造的最小示例。未经双方确认，不要通过公开 Issue 或 Pull Request 上传敏感数据。

## 安全边界

- 本软件处理本地项目文件和用户导入模板，使用者应自行管理电脑权限和备份。
- 在线查询会把必要的查询序列发送到所选第三方服务，使用前应确认其当前条款和所在组织的数据政策。
- 软件生成的序列和判读结果必须由专业人员复核，不能作为实验成功或临床用途保证。
