# GeneSnap Workbench v0.3.3 问题修复报告

## 发布结果

本轮问题在 `v0.3.4` 中修复。实现以用户提供的 `V0.33问题汇总.docx`、真实 LV-037 `.dna` 图谱副本和真实擎科引物订购表副本为验收依据；原始私有资料未写入仓库或安装包。

## 已修复问题

1. 表达/reporter 载体导入改为两种明确模式：默认按左右酶切位点定位并自动计算同源臂，或由用户手动粘贴 F/R 同源臂。原始 0-based 边界不再作为用户输入项。
2. `.dna/.gb/.gbk` 读入后显示文件名、格式、长度和拓扑。未知拓扑会明确区分“文件未提供”和“软件按环状载体处理”，不再误报为空序列。
3. 供应商 Excel 模板统一把合并单元格映射到左上角可写单元格，修复 `MergedCell.value is read-only`。
4. 引物订购模板新增订购日期、Gene ID、NM/转录本号和扩增产物长度字段。shRNA 使用完整 oligo 长度，表达类使用对应 PCR fragment 长度，reporter 记录关联构建长度。
5. 项目总览和详情明确区分内部编号、引物送单号、引物订单号、测序送样号和测序订单号。补录/修改需要说明并保留前后值；完成项目保持只读。
6. 质粒抽提不再自动选择每组第一个克隆。用户必须从真实可用克隆中为每个 Target/构建选择至少一个，同一组可多选。
7. 历史页的事件、原状态和新状态统一显示中文，数据库内部稳定 ID 保持不变。
8. 主 splitter 加宽并可拖动，左右面板不会折叠；动作按钮过多时在独立区域横向滚动，项目表底部滚动条不会越过左侧面板。
9. 关闭状态的下拉框、数字框和日期框不再因鼠标悬停滚轮而改变值；下拉列表展开后仍可正常滚动。
10. shRNA 载体确认默认勾选，但用户仍可取消。

- LV-037 SnapGene 图谱：读取为 `snapgene`、`11316 bp`、`circular`；唯一 NheI 位点切点为 `3803`，唯一 BamHI 位点切点为 `3821`；自动同源臂长度分别为 `18 bp` 和 `20 bp`。

- 两种主窗口尺寸 `1040x680`、`1440x860` 已截图检查分栏、动作区、历史中文和项目表滚动条。

## 使用边界

- 自动序列设计和软件校验不代表湿实验一定成功。
- 实验室改造载体应导入实际使用的图谱并人工确认 protocol，不要用名称相近的公开载体替代。
- CKO/Flox 仍不在当前可运行范围，只保留后续扩展接口。
